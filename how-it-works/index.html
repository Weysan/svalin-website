Technical Overview

How Svalin works

Most AI governance tools operate at the network layer — and miss local AI agents entirely. Svalin operates at the machine level, capturing data flows at the source before they leave the device.

The Invisible Problem

Local AI tools bypass every network control

Engineers increasingly use AI tools that run directly on their local machine and connect directly to internal systems — code repositories, databases, file systems — without routing through any corporate network control point.

API gateways, network proxies, and MDM tools all operate at the network or device management layer. They cannot see what a locally-running AI process sends to an external provider. Even a strict usage policy cannot be enforced or verified without a tool that operates at the machine level.

A typical example

A developer asks their AI assistant to optimise a database query. Without anyone realising it, the AI tool reads internal files, queries database schemas, and pulls relevant Jira tickets — then bundles everything and sends it to an external AI provider's servers. The developer sees a helpful answer. No one sees what left the building.

The Process

Four steps from invisible risk to full control

1

See everything — including what bypasses the network

Svalin's agent operates at the machine level, capturing every AI tool data transfer before it reaches an external AI provider. Covers what API gateways, proxies, and MDM tools all miss — local AI agent access to Jira, BigQuery, Google Drive, databases, code repositories, and internal APIs.

2

Set the rules, enforce them automatically

Define which data categories — PII, credentials, financial data, health data, confidential IP — can flow through which AI tool connections. The policy engine warns or blocks in real time, without interrupting your team's workflow. Policies are version-controlled with a full change history.

3

Prove compliance when asked

Export structured audit reports mapped to GDPR, EU AI Act, SOC 2, ISO 27001, ISO 42001, or NIST AI RMF. Every policy decision and governance action is logged with full context — user, data categories, AI provider, timestamp, policy applied.

4

Zero disruption for your team

The local agent never interferes with AI tool calls. Employees keep working at full speed. If Svalin goes down, nothing changes for end users — the agent simply stops logging. There is no inline blocking that could cause downtime.

Current Limitations

A note on scope

Important to know

Svalin captures data flows through custom connections that engineers configure — the connections to Jira, BigQuery, Google Drive, and similar tools described above. Built-in capabilities of AI tools themselves — such as Claude's native web browsing or built-in code execution — are not currently observable, as AI providers do not expose that data externally. This is an industry-wide limitation today, not specific to Svalin. The custom connection layer we monitor is where the majority of sensitive organisational data flows.
