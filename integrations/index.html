Data Sources

Jira & project tools

When AI tools access Jira for context — tickets, sprint data, project metadata — that information gets bundled into requests sent to external AI providers. Svalin captures exactly which Jira data was accessed and when.

BigQuery & databases

Database schemas, query results, and table data accessed by AI coding assistants represent some of the highest-risk data flows. Svalin logs what was queried, what was returned, and what was sent to the AI provider.

Google Drive & documents

Documents, spreadsheets, and files accessed through Google Drive connections to AI tools — including sensitive internal documents that should not be leaving your organisation.

GitHub & code repositories

AI coding assistants reading your codebase for context send proprietary code to external AI providers. Svalin makes that flow visible and governable — capturing what code was accessed and what was sent.

Local file systems

File system access is the most direct path for sensitive documents to reach external AI providers. Svalin captures file system MCP server calls including file paths and content categories accessed.

Technical Note

How integrations work

Svalin monitors data flows at the MCP (Model Context Protocol) layer — the protocol that connects AI tools to external data sources. When an AI tool makes a tool call to a connected system (Jira, BigQuery, Google Drive, etc.), that call passes through the MCP layer. Svalin's agent captures these calls at the machine level, before the data reaches the AI provider.

This means no changes are required to your existing integrations, cloud configuration, or network setup. The agent installs on developer machines and immediately begins capturing data flows from all configured MCP server connections.
