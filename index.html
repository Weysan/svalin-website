Features

Everything you need to govern AI

From audit trails to policy enforcement, Svalin gives security and compliance teams full control over AI usage.

Svalin deploys a lightweight agent to developer machines through your MDM — Jamf, Intune, or Kandji. The agent auto-detects supported AI coding tools and wraps their MCP configurations to proxy tool calls. It also configures itself as an LLM proxy where supported. Process tree analysis attributes agent activity to specific developer sessions, allowing Svalin to reconstruct the full coding flow from developer prompt to session close — without touching the developer's workflow. Telemetry is shipped to an append-only platform backend. Svalin's scope is the developer tool layer — MCP tool calls, LLM interactions, and configuration integrity for supported agents. Activity outside that scope is outside its visibility.

How it works in detail →

Tamper-Evident Audit Chain

Every AI session on enrolled devices is logged with a cryptographic chain that proves log integrity and ordering. Configuration tampering and known bypass attempts are detected and alerted — giving auditors tamper-evident evidence of observed activity and device posture.

Full Session Visibility

MCP tool calls and LLM interactions for supported agents — Claude Code, Claude Desktop, Cursor, Gemini CLI — captured at the device level without requiring gateway routing. Svalin's scope is the developer tool layer; activity outside supported agents is outside its visibility.

Policy Engine

Deploy and enforce AI usage policies from your MDM — Jamf, Intune, or Kandji. Define which data categories can flow to which AI providers. Policies apply on device, before traffic leaves the machine.

Multi-Model Support

Works across Claude Code, Claude Desktop, Cursor, and Gemini CLI today, with support for additional MCP-compatible agents on the roadmap. One governance layer for your supported AI stack.

Bypass Detection

Svalin continuously monitors agent configuration files for tampering — unapproved MCP servers, proxy URL changes, hook suppression. Detected changes trigger automatic policy re-application and a compliance alert on the platform. Known bypass paths on enrolled devices are surfaced in real time.

Frequently asked questions

Quick answers to the most common questions about Svalin.

All data is fully encrypted and stored on European servers. Encryption keys are hosted with a separate European cloud provider and are never shared with our hosting platform, ensuring complete separation of concerns.

Logging scope is configurable. By default, Svalin logs session metadata — data categories transferred, AI provider, timestamp, and policy decisions — not raw conversation content.
